# 128. 最长连续序列

## 题目描述

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 **O(n)** 的算法解决此问题。

**示例：**

```
输入：nums = [100, 4, 200, 1, 3, 2]
输出：4
解释：最长连续序列是 [1, 2, 3, 4]，长度为 4
```

## 思路分析

### 核心思想：用哈希集合去重 + 只从序列起点开始查找

暴力做法是对每个元素，向左/右扩张找连续序列，时间复杂度 O(n²)。要优化到 **O(n)**，关键有两点：

1. **用集合去重** — O(1) 的查找时间。
2. **只从序列的起点开始计数** — 避免重复计算。

#### 如何判断起点？

一个元素 `x` 是某个连续序列的起点 $\iff$ **`x - 1` 不在集合中**。

- `x = 1`，`0` 不在集合中 $\rightarrow$ 起点，向后找 `2, 3, 4`，长度 4 ✅
- `x = 4`，`3` 在集合中 $\rightarrow$ 不是起点，跳过 ❌（4 已经在以 1 为起点的序列中被数过了）

这样每个元素最多被访问 **两次**（一次查起点，一次向后延伸），整体 O(n)。

### 算法步骤

1. 将所有数字放入 `set`。
2. 遍历 `nums`：
   - 如果 `x - 1` 不在集合中（即 `x` 是序列起点）：
     - 从 `x` 开始，不断检查 `x + 1, x + 2, ...` 是否在集合中，统计连续长度。
     - 更新全局最大值。
   - 否则跳过。

## Python 代码

```python
def longestConsecutive(nums: list[int]) -> int:
    num_set = set(nums)
    longest = 0

    for x in num_set:          # 遍历 set 而非 nums，去重后少一些不必要的判断
        # 只从序列起点开始查找
        if x - 1 not in num_set:
            cur = x
            length = 1

            while cur + 1 in num_set:
                cur += 1
                length += 1

            longest = max(longest, length)

    return longest
```

## 复杂度分析

| 时间复杂度 | 空间复杂度 |
|-----------|-----------|
| **O(n)** | **O(n)** |

- **时间**：每个元素进入 `while` 循环的次数最多 1 次（只有作为序列起点时才会进入），所以整体是线性扫描。
- **空间**：需要一个哈希集合存储所有数字。

## 易错点

1. **遍历 set 还是 nums？** 都可以，但遍历 set 可以避免重复元素导致的无意义判断。不过不影响复杂度。
2. **`x - 1 in set` 的判断必须在遍历 set 时完成** — 如果是遍历原始的 nums，重复元素会导致多次从同一起点出发，虽然不影响结果但浪费时间。
3. **空数组处理**：`num_set` 为空时，`for` 循环不会执行，返回 0，正确。
