# 15. 三数之和

## 题目描述

给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。

请你返回所有和为 `0` 且 **不重复** 的三元组。

**注意：** 答案中不可以包含重复的三元组。

**示例：**

```
输入：nums = [-1, 0, 1, 2, -1, -4]
输出：[[-1, -1, 2], [-1, 0, 1]]
```

## 思路分析

### 核心思想：排序 + 双指针

暴力三重循环 O(n³) 会超时，需要优化。

关键优化点：
1. **排序** — 让数组有序，便于去重和双指针。
2. **固定一个数 + 双指针找另外两个** — 将三数之和转化为两数之和问题。

### 算法步骤

1. 对数组 **排序**。
2. 遍历数组，固定 `nums[i]` 作为第一个数：
   - **去重**：如果 `i > 0` 且 `nums[i] == nums[i-1]`，跳过（避免重复三元组）。
   - **剪枝**：如果 `nums[i] > 0`，直接 break（排序后后面的数都 $\ge$ `nums[i]`，三数和不可能为 0）。
   - 用双指针 `l = i + 1`，`r = len(nums) - 1` 在 `[i+1, n-1]` 范围内找和为 `-nums[i]` 的两个数：
     - `s = nums[l] + nums[r]`
     - 如果 `s < target`：左指针右移（需要更大的数）
     - 如果 `s > target`：右指针左移（需要更小的数）
     - 如果 `s == target`：找到一个解，记录，然后 **移动双指针并去重**：
       - `l++` 并跳过重复的 `nums[l]`
       - `r--` 并跳过重复的 `nums[r]`

### 为什么排序 + 双指针不会漏解？

排序后，固定 `nums[i]`，在 `[i+1, n-1]` 中找两数之和为 `target`。双指针从两端向中间移动：
- `l` 右移 $\rightarrow$ 和增大
- `r` 左移 $\rightarrow$ 和减小

每次比较 `sum` 与 `target` 的大小关系，移动一个指针，**不会漏掉任何可能的配对**（类似第 11 题盛水容器的证明思路）。

## Python 代码

```python
def threeSum(nums: list[int]) -> list[list[int]]:
    nums.sort()
    n = len(nums)
    res = []

    for i in range(n - 2):
        # 剪枝：最小的数已经 > 0，后面不可能有解
        if nums[i] > 0:
            break

        # 去重：跳过重复的第一个数
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        l, r = i + 1, n - 1
        target = -nums[i]

        while l < r:
            s = nums[l] + nums[r]

            if s < target:
                l += 1
            elif s > target:
                r -= 1
            else:
                # 找到一个三元组
                res.append([nums[i], nums[l], nums[r]])

                # 去重：跳过重复的第二个数
                while l < r and nums[l] == nums[l + 1]:
                    l += 1
                # 去重：跳过重复的第三个数
                while l < r and nums[r] == nums[r - 1]:
                    r -= 1

                l += 1
                r -= 1

    return res
```

## 复杂度分析

| 时间复杂度 | 空间复杂度 |
|-----------|-----------|
| **O(n²)** | **O(1)**（不计返回值的空间）|

- 排序 O(n log n) + 双指针遍历 O(n²) = O(n²)。
- 额外空间只用了几个指针变量（排序的递归栈空间不计入）。
- 相比于暴力 O(n³)，优化了一个数量级。

## 易错点

1. **去重的时机**：要在找到解之后去重，而不是在移动指针之前去重 — 否则会漏解。
2. **剪枝条件**：`nums[i] > 0` 才 break，不是 `nums[i] >= 0`。因为 `nums[i]` 可以是 0（如 `[0, 0, 0]`）。
3. **`i < n - 2`**：至少要留两个位置给 `l` 和 `r`。
4. **`target = -nums[i]`**：将三数和为零转为两数和为 `-nums[i]`，逻辑更清晰。
