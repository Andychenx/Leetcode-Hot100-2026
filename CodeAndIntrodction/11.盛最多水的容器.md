# 11. 盛最多水的容器

## 题目描述

给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。

找出其中的两条线，使得它们与 `x` 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

**说明：** 不能倾斜容器。

**示例：**

```
输入：[1,8,6,2,5,4,8,3,7]
输出：49
解释：选择 height[1]=8 和 height[8]=7，距离为 7，面积 = min(8,7) * 7 = 49
```

## 思路分析

### 核心思想：双指针 + 贪心移动

容器的盛水量由 **较短的边** 和 **两条线之间的距离** 决定：

```
面积 = min(height[l], height[r]) × (r - l)
```

使用两个指针 `l` 和 `r` 分别指向数组两端，每次计算当前面积并更新最大值。

关键决策：**移动哪一侧的指针？**

- 如果移动 **较长** 的那一侧：宽度减少，高度被短板决定，**面积不可能变大**。
- 如果移动 **较短** 的那一侧：宽度减少，但 **有机会遇到更高的板子**，从而可能增大面积。

$\Rightarrow$ 每次移动 **较矮的那一侧指针**。

### 正确性证明

假设 `height[l] < height[r]`，当前面积 `= height[l] × (r - l)`。

如果固定 `l`，无论 `r` 怎么左移，宽度都在缩小，且高度 $\le$ `height[l]`（被短板限制），面积只会更小。所以 `l` 这个位置 **不可能成为最优解的左边界**，可以安全地向右移动。

## Python 代码

```python
def maxArea(height: list[int]) -> int:
    l, r = 0, len(height) - 1
    ans = 0

    while l < r:
        # 计算当前面积
        area = min(height[l], height[r]) * (r - l)
        ans = max(ans, area)

        # 移动较矮的那一侧
        if height[l] < height[r]:
            l += 1
        else:
            r -= 1

    return ans
```

## 复杂度分析

| 时间复杂度 | 空间复杂度 |
|-----------|-----------|
| **O(n)** | **O(1)** |

- 两个指针从两端向中间遍历，每个元素最多访问一次。
- 只用了几个整型变量，没有额外空间。

## 易错点

1. **移动条件**：是 `height[l] < height[r]` 时移左，不是 `area` 的大小关系。
2. **相等时**：两边一样高时移哪边都可以，因为无论移哪边，当前的 `l` 和 `r` 都不可能是更优解了（被另一侧等高的板子替代）。
3. **面积公式**：高度取 `min`，不是 `max` — 水会从矮的那边溢出。
